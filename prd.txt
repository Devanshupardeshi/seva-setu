# SevaSetu — Full Product Requirements Document (PRD)

*Google Solution Challenge 2026 — Track: Smart Resource Allocation (Data-Driven Volunteer Coordination for Social Impact)*

---

## 1. Executive Summary

**Product name:** SevaSetu
**Tagline:** "The bridge between willing hands and urgent needs."
**One-liner:** An AI-powered, multilingual, real-time platform that matches volunteers to NGOs and disaster-response needs using Gemini, Vertex AI vector search, and Google Maps — deployed on GCP.

**North-star metric:** *Hours of meaningful volunteer work mobilized per week.*
**Winning thesis:** Follow the proven Saheli + HeadHome blueprint (Flutter + Firebase + Gemini + Maps) but add a **disaster-response command center** that no previous finalist has built.

---

## 2. Problem Statement & Goals

### Problem

1. India has 33 lakh NGOs but zero unified volunteer infrastructure.
2. Volunteer drop-off rate is ~70% within 3 months due to bad matches.
3. During disasters, coordination happens over WhatsApp; hours are lost.
4. Small NGOs have no HR capacity to filter applicants.


### Goals (SMART)

| Goal | Target by Demo Day
|-----|-----
| Onboard pilot NGOs | 10 partner NGOs
| Volunteer profiles on platform | 500 real users
| Successful matches made | 100
| Languages supported | 5 Indian languages + English
| Disaster drill simulation | 1 completed live drill with a state NDRF / NSS chapter


### SDGs Addressed

- **SDG 3** — Good Health and Well-Being
- **SDG 11** — Sustainable Cities and Communities
- **SDG 17** — Partnerships for the Goals
- Secondary: SDG 4 (Education), SDG 5 (Gender Equality), SDG 10 (Reduced Inequalities)

## 4. Scope — MVP vs Phase 2 vs Long-term

### MVP (what we ship for Demo Day — 10 weeks)

1. Volunteer mobile app (Flutter) with voice onboarding
2. NGO web dashboard (Next.js)
3. AI matching engine (Vertex AI Vector Search)
4. Push notifications + briefings (Gemini)
5. Disaster command center dashboard (web, live map)
6. Basic trust score + reviews
7. 5 Indian languages


### Phase 2 (post-selection, Phase 2 of Solution Challenge)

- CSR corporate portal
- Government integration (NDMA / NSS APIs)
- Offline-first disaster mode (MapsSDK tiles cached)
- WhatsApp bot via Business API
- Certificate NFT-free verification (Cloud Signing)


### Long-term Vision

- Default volunteer infrastructure for state governments
- Cross-border NGO mobility (SAARC region)
- Integration with CoWIN-like citizen services layer

### 6.2 Three-tier breakdown

**Presentation Tier**

- Flutter (Android first, iOS later) — volunteer app
- Next.js 16 + App Router + shadcn/ui — NGO + Disaster dashboards
- Hosted on Firebase Hosting / Vercel


**Application Tier**

- Cloud Run services (containerized Node/TS microservices)
- Cloud Functions for lightweight event handlers
- Pub/Sub as the event backbone (matches, mobilization, reviews)
- API Gateway for unified routing and auth


**Data Tier**

- Firestore — operational DB (users, NGOs, needs, matches)
- BigQuery — analytics warehouse (mirrored from Firestore)
- Cloud Storage — voice intros, profile pics, generated PDFs
- Vertex AI Vector Search Index — semantic matching


**AI Tier**

- Gemini 2.5 Pro / Flash — multimodal NLP, briefings, parsing
- Vertex AI `text-embedding-004` — 768-dim embeddings
- Vertex AI Vector Search — sub-second nearest-neighbor on 500K+ vectors
- Speech-to-Text — fallback for non-Gemini languages

## 9. GCP Service Mapping (what we use and why)

| Service | Purpose | Justification
|-----|-----
| **Firebase Auth** | Phone OTP auth | Zero friction, India-ready
| **Firestore** | Operational DB | Real-time sync for dashboards
| **Cloud Run** | Stateless API services | Autoscales to zero; cheap
| **Cloud Functions** | Triggers on Firestore / Storage | Simple glue
| **Pub/Sub** | Event bus | Decouples match, notify, analytics
| **Cloud Storage** | Audio, PDFs, images | Cheap, signed URLs
| **BigQuery** | Analytics and impact reporting | Judges love SQL-able impact metrics
| **Vertex AI (Gemini 2.5)** | NLP + briefings | Multilingual Indian contexts
| **Vertex AI Vector Search** | Semantic matching | Sub-second on 500K+ vectors
| **Vertex AI Embeddings** | Create vectors | Required for semantic match
| **Maps Platform** | Routing, geofencing, heatmaps | Core to disaster view
| **Places API** | NGO / venue lookup | Saves NGOs typing addresses
| **Looker Studio** | Public impact dashboard | Embeds in pitch website
| **Firebase Cloud Messaging** | Push | Free and reliable
| **Firebase App Check** | Anti-abuse | Required for trust score integrity

## 10. API Contract (core endpoints)

| Endpoint | Method | Purpose
|-----|-----
| `/volunteers/onboard` | POST (multipart audio) | Upload voice → Gemini extract
| `/volunteers/:id/profile` | GET/PATCH | Fetch/edit profile
| `/ngos/register` | POST | Darpan verification
| `/needs` | POST | NGO posts a need
| `/needs/:id/matches` | GET | Ranked matches list
| `/matches/:id/accept` | POST | Volunteer accepts, triggers briefing
| `/matches/:id/briefing` | GET | AI briefing doc
| `/incidents` | POST | DM office creates disaster
| `/incidents/:id/mobilize` | POST | Mass push to matched volunteers
| `/incidents/:id/dashboard` | GET (SSE) | Live heatmap stream
| `/impact/:userId` | GET | Hours, people helped, certs

## 11. Build Plan — 10-Week Roadmap

### Week 1 — Foundations

- Create GCP project, enable APIs, set up Firebase project
- Define Firestore schema, create security rules
- Set up Cloud Run scaffolding + CI/CD via GitHub Actions
- Design system (Figma) and brand identity
- Register for Darpan API access


### Week 2 — Auth & Profiles

- Flutter: phone OTP + volunteer profile screens
- Next.js: NGO dashboard shell + Darpan signup
- Firestore user + ngo collections live


### Week 3 — Voice Onboarding (the wow-moment)

- Record audio in Flutter → upload to Cloud Storage
- Cloud Function → Gemini 2.5 multimodal prompt → structured JSON
- Confirmation screen for user
- Embedding generation and Vector Search insert


### Week 4 — NGO Posting Flow

- Natural-language need input
- Gemini parses to structured JSON
- Embedding + Vector Search query
- Ranked applicant list UI


### Week 5 — Matching + Briefings

- Match creation service
- Gemini briefing generator (prompt tuning)
- FCM push + deep link to match detail


### Week 6 — Disaster Command Center

- Incident creation form
- Live map with Maps Platform Deck.gl / Heatmap layer
- Bulk mobilization endpoint
- SSE stream of live acceptances


### Week 7 — Trust, Safety, Certificates

- Review + rating post-match

- PDF certificate generation (Puppeteer on Cloud Run)
- Verified NGO badge


### Week 8 — Multilingual + Analytics

- Gemini prompts tested for Hindi, Marathi, Tamil, Telugu, Bengali

- Looker Studio public impact dashboard

- Run a 1-day disaster simulation drill
