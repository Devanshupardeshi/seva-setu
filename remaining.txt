**Collections to create (from PRD §7):**

**Security rules** (critical — put in `firestore.rules`):

```plaintext
match /users/{uid} {
  allow read: if request.auth != null;
  allow write: if request.auth.uid == uid;
}
match /ngos/{ngoId} {
  allow read: if true;
  allow write: if request.auth != null &&
    get(/databases/$(database)/documents/ngos/$(ngoId)).data.coordinators[request.auth.uid] == true;
}
match /matches/{matchId} {
  allow read: if request.auth.uid == resource.data.userId ||
    get(/databases/$(database)/documents/ngos/$(resource.data.ngoId)).data.coordinators[request.auth.uid] == true;
}
```

### Integration 3 — Vertex AI Vector Search (the matching engine)

**Why:** This is the 40%-weight technical-merit anchor. Currently `app/ngo/needs/[id]/page.tsx` uses hardcoded `rankedApplicants`.

**Setup (one time):**

1. Enable Vertex AI API in GCP
2. Create a Vector Search Index:

1. Dimensions: **768** (for `text-embedding-004`)
2. Distance metric: **cosine**
3. Shard size: small (free tier)



3. Deploy the index to an endpoint
4. Install: `pnpm add @google-cloud/aiplatform`


**New files to create:**

```typescript
// lib/ai/embeddings.ts
import { PredictionServiceClient } from "@google-cloud/aiplatform"

const client = new PredictionServiceClient({
  apiEndpoint: "us-central1-aiplatform.googleapis.com",
})

export async function embed(text: string): Promise<number[]> {
  const [response] = await client.predict({
    endpoint: `projects/${process.env.GCP_PROJECT}/locations/us-central1/publishers/google/models/text-embedding-004`,
    instances: [{ structValue: { fields: { content: { stringValue: text } } } }],
  })
  return response.predictions?.[0]?.structValue?.fields?.embeddings?.structValue
    ?.fields?.values?.listValue?.values?.map((v: any) => v.numberValue) ?? []
}
```

```typescript
// lib/ai/vector-search.ts
import { MatchServiceClient } from "@google-cloud/aiplatform"

const match = new MatchServiceClient({
  apiEndpoint: `${process.env.VECTOR_INDEX_ENDPOINT_DOMAIN}`,
})

export async function findNearest(vector: number[], topK = 10) {
  const [res] = await match.findNeighbors({
    indexEndpoint: process.env.VECTOR_INDEX_ENDPOINT!,
    deployedIndexId: process.env.VECTOR_INDEX_DEPLOYED_ID!,
    queries: [{ datapoint: { featureVector: vector }, neighborCount: topK }],
  })
  return res.nearestNeighbors?.[0]?.neighbors ?? []
}
```

**Pipeline:**

1. When volunteer completes onboarding → Cloud Function creates their embedding from the structured profile → upserts to vector index
2. When NGO posts a need → embed the need → `findNearest` returns top 10 volunteer IDs
3. Re-rank in Node: weight by `distance × 0.6 + trustScore × 0.2 + geoProximity × 0.2`
4. Return to `/ngo/needs/[id]` applicants page


**Files to touch:**

- `app/api/match/route.ts` (new) — the matcher endpoint
- `app/ngo/needs/[id]/page.tsx` — fetch from `/api/match?needId=...` instead of mock

### Integration 4 — Gemini (3 separate prompts)

**Why:** Three distinct AI moments in the demo:

- (a) Voice onboarding extraction
- (b) NGO need parsing
- (c) Match briefing generation


**Setup:**

1. Enable Vertex AI → Gemini API
2. Install: `pnpm add @google/generative-ai` (public API) OR `@google-cloud/vertexai` (production)
3. Add key: `GEMINI_API_KEY` or use service account for Vertex


**Prompt A — Voice onboarding** (replaces simulated extraction in `components/volunteer/voice-recorder.tsx` + `extraction-stream.tsx`)

```typescript
// app/api/onboarding/extract/route.ts
import { GoogleGenerativeAI } from "@google/generative-ai"

export async function POST(req: Request) {
  const form = await req.formData()
  const audio = form.get("audio") as File

  const genAI = new GoogleGenerativeAI(process.env.GEMINI_API_KEY!)
  const model = genAI.getGenerativeModel({ model: "gemini-2.5-flash" })

  const audioBytes = Buffer.from(await audio.arrayBuffer()).toString("base64")

  const result = await model.generateContent([
    { inlineData: { mimeType: "audio/webm", data: audioBytes } },
    {
      text: `Extract volunteer information from this audio recording.
Return strict JSON matching:
{
  "name": string,
  "languages": string[],
  "skills": string[],
  "availability": string[],
  "location": string,
  "experienceLevel": "student" | "professional" | "retired"
}
If a field is unclear, return null. No extra commentary.`,
    },
  ])
  const json = JSON.parse(result.response.text())
  return Response.json(json)
}
```

In `voice-recorder.tsx`, replace the `setTimeout` simulation with:


4. Materialized views aggregate by SDG, region, week


**Next.js query:**

```typescript
// lib/bigquery.ts
import { BigQuery } from "@google-cloud/bigquery"
export const bq = new BigQuery({ projectId: process.env.GCP_PROJECT })

export async function getWeeklyTrend() {
  const [rows] = await bq.query(`
    SELECT
      DATE_TRUNC(completed_at, WEEK) AS week,
      SUM(hours) AS hours,
      COUNT(*) AS matches
    FROM sevasetu_analytics.verified_hours
    WHERE completed_at >= DATE_SUB(CURRENT_DATE(), INTERVAL 12 WEEK)
    GROUP BY week ORDER BY week
  `)
  return rows
}
```
